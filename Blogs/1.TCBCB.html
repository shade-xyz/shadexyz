Memory Traces Xplained (Well the 3 main ones)

So basically in this article, I will explain the three main memory trace topologies on Motherboards.

Why should you care? Well, it’s a good indicator of its behavior while memory overclocking. Though this is not the end all be all, So get informed and make an informed decision.

So, The three layouts we are going to be comparing are: One DIMM Per Channel Fly-by or Daisy chain (I’ll be referring to it as daisy chain because I like it) T-Topology

One DIMM per Channel: A very basic and easy-to-understand layout, basically your two memory channels are mapped to two memory slots. That means one memory channel has to communicate with one memory slot.

Daisy chain: Here we see 2 memory slots being connected/mapped to one memory controller and there are generally 4 slots on consumer Motherboards so 4 slots are mapped to 2 controllers. When you do use this layout Generally, you’d have one slot that performs good and the other one that kinda doesn’t, as in one of the slots doesn’t support anywhere as near as high memory speed as the other one. But this doesn’t mean it's going to be a disaster, It's really popular, it's just that it favors one of the slots, and usually, it's the second one but it entirely depends on the manufacturers and their implementation of the said layout.

T-Topology: This one is kinda special the traces that map the two slots to one controller are equidistant so both of the memory slots are essentially equal unlike the Daisy chain neither one of the slots performs better when compared and again usually there are 4 slots mapped to 2 controllers. There is a problem tho, Both of the memory traces from the controller to the slot are longer than the ones in the Daisy chain, so if you are using two sticks of ram on this layout it’s going to be slower than the other two counterparts.
